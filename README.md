## 📐 Health Score Methodology

| Signal | Weight | Scoring Criteria |
|--------|--------|-----------------|
| Commit Recency | 30 pts | Daily=30, Weekly=20, Monthly=10, Older=0 |
| Issue Response Rate | 25 pts | <10 issues=25, <50=15, <100=5, 100+=0 |
| PR Merge Velocity | 25 pts | <7 days=25, <30 days=15, <60 days=5, older=0 |
| Star Momentum | 20 pts | >500/wk=20, >100=15, >10=10, <10=5 |

## 🔄 How It Works

1. Every **Monday**, an automated agent searches GitHub trending for Python, JavaScript, and Rust
2. Live stats are pulled via the GitHub API (stars, issues, PRs, commit timestamps)
3. Weekly star counts are pulled directly from GitHub's trending page (`?since=weekly`)
4. Health Scores are recalculated using the formula above
5. The old README is archived to `/reports/YYYY-MM-DD.md`
6. This dashboard is rewritten with fresh data and committed

Scripts and methodology: [`/scripts/analyze.md`](./scripts/analyze.md)
